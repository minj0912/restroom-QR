React + TypeScript + Firebase Firestore 기반의 화장실 위생점검표 웹앱을 만들어 주세요.

중요:
- 현재 단계는 MVP 개발 단계입니다.
- 배포 전용 최종 보안 작업은 아직 하지 않습니다.
- 현재는 프론트에서 점검자모드/관리자모드 코드를 비교하는 방식으로 구현합니다.
- 시간 제한(오전 11시 / 오후 16시 전 입력 제한)은 아직 구현하지 않습니다. 추후 추가 예정입니다.
- 지금은 “일반 조회 + 점검자 입력 + 관리자 관리”까지 동작하는 구조를 우선 구현해 주세요.
- 스타일은 깔끔한 운영용 대시보드 느낌으로 정리해 주세요.
- 너무 과한 라이브러리 추가는 하지 말고, 현재 구조는 단순하고 유지보수 쉽게 작성해 주세요.

기술 스택:
- React
- TypeScript
- Firebase Firestore
- 현재는 단일 App.tsx 중심으로 구현해도 괜찮습니다
- firebase 패키지만 사용해도 됩니다
- 라우터는 지금 단계에서는 없어도 됩니다
- Tailwind 없이 인라인 스타일 또는 기본 CSS로 구현해도 됩니다

앱 목적:
기존에 화장실 입구에 종이로 붙어 있던 일일 위생점검표를 QR 기반 웹앱으로 대체하려고 합니다.
사용자는 QR 코드로 접속해서 오늘 점검표를 조회할 수 있어야 하고, 이전 날짜 기록도 조회할 수 있어야 합니다.
점검자와 관리자는 상단 버튼을 통해 모드 전환 후 입력/관리 기능을 사용할 수 있어야 합니다.

권한 구조:
1. 일반모드(viewer)
- 조회만 가능
- 오늘 또는 선택한 날짜의 점검표를 읽기 전용으로 볼 수 있어야 함
- 화장실 선택 가능
- 날짜 선택 가능

2. 점검자모드(inspector)
- 코드: 6481
- 점검자명 입력 가능
- 항목별 O / X 선택 가능
- 점검 저장 가능
- 점검표 항목 수정 불가
- 화장실 추가/수정/삭제 불가

3. 관리자모드(admin)
- 코드: 6167
- 점검자 기능 모두 가능
- 점검표 항목 추가 / 수정 / 삭제 가능
- 화장실 추가 / 수정 / 삭제 가능

필수 기능:
1. 상단 헤더
- 제목: 화장실 위생점검표
- 현재 모드 배지 표시: 일반모드 / 점검자모드 / 관리자모드
- 일반모드일 때:
  - 점검자모드 버튼
  - 관리자모드 버튼
- 점검자모드 또는 관리자모드일 때:
  - 일반모드로 돌아가기 버튼

2. 필터 영역
- 날짜 선택 input(type="date")
- 화장실 선택 select
- 기준일과 현재 권한 상태 요약 표시

3. 일반 조회 영역
- 선택된 화장실 정보 표시
- 선택된 날짜의 템플릿 기준으로 오전 점검 / 오후 점검 섹션 표시
- 각 섹션에 상태 배지 표시: 점검 완료 / 미점검
- 점검자명 표시
- 항목별 결과 O/X 또는 미입력(-) 표시

4. 점검자 입력 기능
- 점검자모드 또는 관리자모드일 때만 입력 UI 노출
- 슬롯별 점검자명 입력창
- 항목별 O / X 선택 버튼
- 슬롯별 저장 버튼
- 저장 시 inspectionRecords 문서 생성 또는 업데이트
- 저장 전 유효성 검사:
  - 점검자명 필수
  - enabled=true인 항목은 모두 O/X 선택 필수

5. 관리자 기능 - 점검표 관리
- 관리자모드일 때만 노출
- 현재 템플릿 항목 목록 편집 가능
- 항목 ID 수정 가능
- 항목명 수정 가능
- 순서 수정 가능
- 사용 여부 수정 가능
- 항목 추가 버튼
- 항목 삭제 버튼
- 점검표 저장 버튼
- 저장 시 inspectionTemplates 문서 업데이트

6. 관리자 기능 - 화장실 관리
- 관리자모드일 때만 노출
- 현재 선택된 화장실 정보를 폼에 표시
- 화장실 ID, 이름, 코드, 위치, 정렬 순서 수정 가능
- 새 화장실 입력 버튼
- 화장실 저장 버튼
- 현재 선택 화장실 삭제 버튼
- 저장 시 restrooms 컬렉션 문서 저장
- 삭제 시 restrooms 컬렉션 문서 삭제

UI/스타일 요구사항:
- 전체적으로 깔끔하고 현대적인 운영용 대시보드 느낌
- 흰색 카드 + 연회색/푸른 계열 배경
- 상태 배지, O/X 배지, 버튼이 시각적으로 구분되게
- 모바일에서도 어느 정도 보이되, 우선순위는 데스크탑 웹
- 너무 화려한 애니메이션은 불필요
- 인라인 스타일 또는 간단한 CSS 사용 가능

타입 설계:
가능하면 아래 타입을 기준으로 작성해 주세요.

Restroom:
{
  id: string;
  name: string;
  code: string;
  locationLabel: string;
  enabled: boolean;
  templateId: string;
  sortOrder: number;
}

InspectionSlot:
{
  status: "PENDING" | "DONE";
  inspectorName: string;
  answers: Record<string, "O" | "X">;
  memo: string;
  checkedByRole: "INSPECTOR" | "ADMIN" | null;
}

InspectionRecord:
{
  id: string;
  date: string;
  restroomId: string;
  restroomName: string;
  templateId: string;
  slots: Record<string, InspectionSlot>;
}

InspectionTemplate:
{
  id: string;
  name: string;
  enabled: boolean;
  slots: {
    id: string;
    label: string;
    openHour: number;
    openMinute: number;
  }[];
  items: {
    id: string;
    label: string;
    type: "OX";
    order: number;
    enabled: boolean;
  }[];
}

반드시 포함할 구현 포인트:
- 오늘 날짜를 Asia/Seoul 기준 YYYY-MM-DD로 생성
- inspectionRecords 문서 ID는 반드시 `${date}_${restroomId}` 형식
- restroom 목록은 enabled=true만 불러오기
- sortOrder 기준 오름차순 정렬
- 템플릿 항목은 enabled=true만 일반 조회/점검 UI에서 노출
- 새 화장실 ID 입력 시 sanitize 처리:
  - trim
  - 소문자
  - 공백 -> underscore
  - 영문/숫자/언더스코어/하이픈 외 제거
